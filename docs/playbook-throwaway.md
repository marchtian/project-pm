# Playbook B：可丢弃项目协作剧本

> **适用场景**：原型 / PoC / 玩具 / 学习项目 / 一次性脚本 / 给自己看的小工具。
> **典型时长**：几小时到一天。
> **规模定位**：小应用级（几个文件 / 一个能跑的 demo）。
> **核心原则**：尽量少打扰你；跑起来就行；可以丢。

---

## 一、定位与边界

| 维度 | 取值 |
|------|------|
| 适用项目 | 一次性、不进生产、不交付他人 |
| 设计粒度 | **一次性把所有 Phase 设计完** |
| 人工卡点 | **2 处**（种子 + 一次性设计审查） |
| 测试要求 | 跳过单测；只做最后一次冒烟 |
| 文档要求 | 一份 README + 一份 design.md，**没了** |
| 失败处置 | 只有「完全跑不起来」才停下问你 |
| 提交粒度 | 保留 git commit，但允许大颗粒（一次 commit 全部 OK） |
| ADR / changelog | **不写** |

---

## 二、2 个人工卡点

| # | 位置 | 你做 | 我做 |
|---|------|------|------|
| G1 | Step 1 种子 | 一段话说清想做什么 | 必要时反问 1–2 个最关键的不确定项 |
| G2 | Step 2 一次性设计审查 | 看一遍 design.md，OK / 改 / 推翻 | 给出可执行的全量设计 |

> **过了 G2 之后，我会一路跑到底，直到全部完成或彻底跑不起来才回来找你。**

---

## 三、5 步主流程

### Step 1 — 种子（人工）
- **你说**：「我想要个 X，能 Y 就行，技术随便。」
- **我做**：如果有明显歧义，反问 1–2 个；没有就直接进 Step 2。
- **产出**：脑子里记住，不写文件。

### Step 2 — 一次性设计（我做，G2 审阅）
- **我做**：把整件事拆完——技术栈、目录结构、所有模块、所有数据结构、所有页面，全列一份。
- **你做**：扫一眼，签字「OK」或者改。
- **产出**：`docs/design.md`（**唯一的设计文档**）

### Step 3 — 一路写完（我做，自动）
- **我做**：按 design.md 一路写代码，**不再分 Phase**，不再问你。
- **遇坑**：能自己修就自己修，最多重试 3 次；3 次都搞不定才停下找你。
- **不允许**：偷偷改设计 / 静默跳过失败 / 加超出设计的功能。

### Step 4 — 冒烟（我+你）
- **我做**：把启动命令贴给你，列 3–5 条「打开 X 看到 Y」的最小验证。
- **你做**：跑一下，PASS 就完事，FAIL 我修。

### Step 5 — 收尾
- **我做**：写一个 30 行的 `README.md`（怎么跑、有什么、缺什么），一次 commit 全部代码。
- **结束**。

---

## 四、文档骨架

```
项目根/
├── README.md      ← 30 行：怎么跑、做了啥、已知缺陷
├── docs/
│   └── design.md  ← 一次性写完的全量设计
└── ...（代码）
```

没有 ADR，没有 changelog，没有 requirements.md，没有 phase-*.md。

---

## 五、什么情况下我会停下来找你

只有这些：
1. **完全跑不起来**：连服务都 boot 不了，连续 3 次重试无效。
2. **设计里有自相矛盾**：写到一半发现 A 模块要的东西 B 模块给不出来。
3. **触发了「不可逆 + 我没把握」**：比如要删 / 重写超过半个项目的代码。

**其他一切情况，包括小报错、风格差异、命名争议、性能不理想、UI 不好看，全部我自己处理或留 TODO，不打扰你。**

---

## 六、进度可见性（极简）

**我只在 3 个时刻主动说话**：种子澄清 / 设计审查 / 冒烟验收。中间过程**不汇报**，避免打扰。

**你想看进度时，去这两个地方**：
- `git log --oneline` — 最直接，commit message 反映做到哪了
- 终端里滚动的工具调用条 (`Bash(...)` / `Write(...)`) — 实时

**不写 TodoWrite，不写 changelog，不写 ADR。** 出事才停下找你。

---

## 七、3 条反偷懒口令（你随时可用）

- **「停」** — 我立刻停手，等你下一句。
- **「贴输出」** — 我把刚才跑的命令真实输出贴出来。
- **「重来」** — 当前文件 / 当前模块作废，按你新指令重写。

---

## 七、什么时候**别**用 Playbook B、改用 Playbook A

- 这东西要给别人用 / 进生产
- 你预计会维护超过 2 周
- 涉及钱、用户数据、不可恢复的副作用
- 你自己也不确定要做成什么样（先用 A 把需求理清）

不确定时，**默认 A**。
