# ADR-001: 选择 Steedos 作为开发平台

- **状态**：已采纳
- **日期**：2026-04-27
- **决策者**：项目负责人 + Claude

## 背景

需要从零构建一个企业级项目管理系统（PM），具备项目/任务/工时/文档/报表等典型 PM 模块，并且：
- 后续要扩充 AI 能力
- 开发主要由 Claude（Opus 4.7）协助完成
- 文档要齐备、可跨会话延续上下文

## 候选方案

| 方案 | 优势 | 劣势 |
|---|---|---|
| **Steedos 低代码平台** | 对象/字段/权限/列表/详情页全部元数据驱动；amis 前端可零代码渲染；支持 Webapp 自定义 React 组件；MongoDB + Moleculer 微服务架构成熟 | 团队需熟悉 Steedos 元数据规范 |
| Django/Rails 全栈手写 | 控制力强 | 大量样板代码；权限/列表/详情都要重复造轮子 |
| Salesforce / 钉钉宜搭 | 生态成熟 | 商业授权成本；定制能力受限；不便嵌入自定义 React 组件 |
| n8n / Appsmith | 快速搭建 | 偏工作流/管理后台，业务对象建模能力弱 |

## 决策

采用 **Steedos（华炎魔方）** 作为基础平台。

## 理由

1. **元数据驱动**：YAML 定义对象/字段/视图/权限，AI 编写元数据效率高、出错少
2. **标准 UI 免费送**：列表/详情/表单由 Amis 自动渲染，省去 80% 前端代码
3. **可扩展性强**：通过 `webapps/` 机制可嵌入任意 React 组件（如甘特图）
4. **服务端可编程**：Moleculer 微服务支持触发器/函数，覆盖业务逻辑
5. **开源 + 中文友好**：方便团队 onboarding，文档与社区支持充分
6. **AI 协作友好**：所有元数据均为人类可读文本（YAML/JSON/JS），Claude 可直接生成与修改

## 影响

- 项目结构遵循 Steedos package 规范（见 `steedos-package-format` 技能）
- 自定义 UI 通过 amis 组件或 webapps 实现，避免脱离平台
- 数据持久化使用 MongoDB（元数据 + 业务数据共用）
- 部署需 MongoDB + Redis 基础设施

## 备注

AI 能力（如智能任务分配、风险预警）预留接口，将在后续阶段通过 Moleculer service + LLM API 集成。
